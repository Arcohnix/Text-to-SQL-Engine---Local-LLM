# Text to SQL — Local LLM

Ask questions in plain English. SQL is generated by a local LLM and executed against your database.  
**Zero hardcoding — works with any schema, any supported database, on any PC.**

---

## Quick start

### 1. Install Ollama
Download from https://ollama.com and start it:
```bash
ollama serve
ollama pull llama3        # or: ollama pull sqlcoder
```

### 2. Install Python dependencies
```bash
pip install -r requirements.txt
```

### 3. Configure your database
Edit **`config.yaml`** — it's the only file you need to touch:

```yaml
database:
  type: sqlite          # sqlite | postgresql | mysql
  path: sample.db       # path to your .db file (SQLite)

llm:
  model: llama3         # any model from `ollama list`
```

For PostgreSQL or MySQL, uncomment and fill in the host/port/name/user/password fields.

### 4. (Optional) Create a sample database
```bash
python create_sample_db.py
```
This creates `sample.db` with customers, products, orders, and order_items tables.

### 5. Run the app
```bash
streamlit run ui/app.py
```

---

## Using your own database
1. Point `config.yaml` → `database.path` to your existing `.db` file (SQLite), or fill in the connection details for PostgreSQL/MySQL.
2. The app auto-discovers all tables and columns — no code changes needed.

## Project structure
```
text_to_sql/
├── config.yaml              ← Edit this (only this)
├── requirements.txt
├── create_sample_db.py      ← Optional sample data
├── core/
│   ├── config.py            ← Loads config.yaml
│   ├── schema.py            ← Auto-discovers DB schema
│   ├── llm.py               ← Talks to Ollama
│   └── executor.py          ← Runs SQL safely
└── ui/
    └── app.py               ← Streamlit interface
```

## Supported databases
| Database   | Driver (auto-installed)     |
|------------|-----------------------------|
| SQLite     | built-in (no extras needed) |
| PostgreSQL | `pip install psycopg2-binary`|
| MySQL      | `pip install pymysql`        |

## Safety
- By default, only `SELECT` queries are allowed (`readonly: true` in config).
- Results are capped at `max_rows` (default 500) to avoid memory issues.
- Both settings are configurable in `config.yaml`.
